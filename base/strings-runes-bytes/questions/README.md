# 字符串、Rune 和字节测验

## 以下哪个字节切片等于 "keeper" 字符串？
```go
// 以下是 "keeper" 各字符对应的码点：
// k => 107
// e => 101
// p => 112
// r => 114
```
1. []byte{107, 101, 101, 112, 101, 114} *正确答案*
2. []byte{112, 101, 101, 112, 114, 101}
3. []byte{114, 101, 112, 101, 101, 112}
4. []byte{112, 101, 101, 114, 107, 101}

## 这段代码会打印什么？
```go
// 码点：
// g => 103
// o => 111
fmt.Println(string(103), string(111))
```
1. 103 111
2. g o *正确答案*
3. n o
4. "103 111"

## 这段代码会打印什么？
```go
const word = "gökyüzü"
bword := []byte(word)

// ö => 2 字节
// ü => 2 字节
fmt.Println(utf8.RuneCount(bword), len(word), len(string(word[1])))
```
1. 7 10 2 *正确答案*
2. 10 7 1
3. 10 7 2
4. 7 7 1

## 以下哪项是正确的？
1. for range 循环遍历字符串的字节
2. for range 循环遍历字符串的 rune *正确答案*

## 对于 UTF-8 编码的字符串值，以下哪项是正确的？
1. rune 总是开始和结束于相同的索引位置
2. rune 可能开始和结束于不同的索引位置 *正确答案*
3. 字节可能开始和结束于不同的索引位置

## 为什么不能修改字符串值的字节？
1. 字符串值是不可变的字节切片
2. 字符串被大量使用，因此在底层它们是共享的
3. 以上所有 *正确答案*
