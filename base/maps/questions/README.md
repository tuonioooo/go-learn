# Go 语言 Map 相关知识测验

## Map 的主要用途是什么？
例如，下面是一个低效的程序，使用循环在数百万个元素中查找：
```go
millions := []int{/* 数百万个元素 */}
for _, v := range millions {
    if v == userQuery {
        // 执行某些操作
    }
}
```
1. Map 允许在 O(1) 时间内快速查找键 *正确*
2. Map 允许在 O(n) 时间内快速查找键
3. Map 允许在 O(1) 时间内快速遍历键

> **1:** 正确。Map 的平均查找时间复杂度为 O(1)。
>
> **2:** Map 的键查找不是 O(n) 时间复杂度。

## 什么时候不应该使用 Map？
1. 需要通过键查找元素时
2. 需要遍历 Map 的键时 *正确*
3. 需要向程序添加结构化数据时

> **1:** 这正是使用 Map 的场景。
> 
> **2:** 正确！遍历 Map 键的时间复杂度是 O(n)。因此，Map 是键遍历最差的数据结构。
>
> **3:** Map 不允许向程序添加结构化数据。

## 以下哪种类型不能作为 Map 的键？
1. map[string]int
2. []string
3. []int
4. []bool
5. 以上所有 *正确*

> **5:** 切片、Map 和函数值是不可比较的。因此，它们不能作为 Map 的键。

## 下面 Map 的键和元素类型分别是什么？
```go
map[string]map[int]bool
```
1. 键: string 元素: bool
2. 键: string 元素: int
3. 键: string 元素: map[int]
4. 键: string 元素: map[int]bool *正确*

> **4:** 这个 Map 包含其他 Map。Map 的元素类型可以是任何类型。

## Map 值在底层是什么？
1. 一个 Map 头
2. 一个指向 Map 头的指针 *正确*
3. 包含三个字段的小型数据结构：指针、长度和容量

> **2:** 正确。Map 是复杂的数据结构。然而，每个 Map 值只是一个指向 Map 头（这是一个更复杂的数据结构）的指针。
